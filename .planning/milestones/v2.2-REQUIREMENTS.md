# Requirements: v2.2 Match Page Rewrite

**Defined:** 2026-02-03
**Core Value:** Match pages must be clean, readable, and optimized for AI citations with zero duplicate content.

## v2.2 Requirements

Requirements for match page rewrite. Each maps to roadmap phases.

### Layout

- [x] **LAYT-01**: Match page renders as single scrollable page (no tabs on any device)
- [x] **LAYT-02**: Score displays exactly once in a single hero section
- [x] **LAYT-03**: Layout adapts to match state (Upcoming/Live/Finished)
- [x] **LAYT-04**: Sections follow consistent order: Match Info -> Narrative -> Predictions -> FAQ
- [x] **LAYT-05**: Mobile layout identical to desktop (no hidden/collapsed content)

### Content

- [x] **CONT-01**: Match hero shows teams, competition, kickoff time, and score (if applicable)
- [x] **CONT-02**: Pre-match narrative renders for upcoming matches
- [x] **CONT-03**: Post-match narrative renders for finished matches
- [x] **CONT-04**: Predictions table shows all 35 models in sortable format
- [x] **CONT-05**: Finished match predictions table shows actual result and points earned
- [x] **CONT-06**: FAQ section auto-generates 5-7 questions from match data
- [x] **CONT-07**: Live match shows current score and match time

### SEO/GEO

- [x] **SGEO-01**: Single consolidated JSON-LD @graph with SportsEvent + FAQPage
- [x] **SGEO-02**: FAQPage schema contains same questions as visible FAQ section
- [x] **SGEO-03**: eventStatus correctly maps to all states (scheduled/live/finished/postponed/cancelled)
- [x] **SGEO-04**: Content uses semantic H2/H3 headings matching FAQ questions
- [x] **SGEO-05**: Answer-first paragraph structure (key info in opening sentences)

### Architecture

- [x] **ARCH-01**: Single MatchDataProvider context supplies data to all components
- [x] **ARCH-02**: No component fetches match data independently (single source of truth)
- [x] **ARCH-03**: Match state derived once at page level, passed via context
- [x] **ARCH-04**: Deprecated components removed (match-header.tsx, tab components)

## Out of Scope

Explicitly excluded. Documented to prevent scope creep.

| Feature | Reason |
|---------|--------|
| Tabs on mobile | User explicitly rejected; hides content from AI crawlers |
| Sticky header | Removed in v2.1; user prefers natural scroll |
| H2H section | Removed in v2.1; not core value |
| League standings | Removed in v2.1; clutters page |
| Multiple score displays | Primary problem to solve |
| Collapsible sections | Hides content from crawlers |
| Swipe gestures | Only needed for tabs, which are excluded |
